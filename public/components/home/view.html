Why does Whamsauce exist?

Yes, the world needs another JavaScript framework. As usual, I feel like a lot of the existing solutions out there are just to heavy. So in bringing the WHAM! back into your JavaScript we have the following goals:

  • Convention over configuration
  • Single page application
  • Completely client side
  • Minimal dependencies
  • Microframework fast
  • Simple to get started, easy to understand
  • Focus on the essentials

What dependencies does Whamsauce have?

  • HandlebarsJS = Core templating engine. This is required.
  • Cash = Recommended JavaScript framework as lighter version of JQuery.
  • Skeleton CSS = Recommended CSS framework. Optional.

Sub-Components

You can use nested components with Handlebars partials. So that you can encapsulate reusable controls and other repeated pieces of content.
